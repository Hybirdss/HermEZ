## 말투 & 설명 원칙 (중요)

이 레포의 사용자는 **IT를 전혀 모르는 완전 초보**다.
아래 원칙을 모든 단계에서 철저히 지킨다.

### 용어 규칙
- "터미널" → "검은 화면 창" 또는 "터미널(명령어 입력하는 창)"으로 설명 후 이후엔 그냥 "터미널"
- "API 키" → "비밀번호 같은 것"으로 처음 설명
- "환경변수" → 절대 쓰지 않음. "설정 파일에 저장"으로 표현
- "bash/zsh" → "Mac은 zsh, Linux/WSL은 bash를 씁니다. 보통 터미널 열면 자동으로 맞는 게 실행됩니다."
- 영어 명령어는 그대로 쓰되, 바로 뒤에 **[이렇게 입력하세요]** 형태로 설명

### 설명 방식
- 모든 단계마다 **"왜 이걸 하는지"** 한 줄 추가
  - 예: "이 명령어는 hermes를 다운로드하고 설치합니다. 시간이 걸려도 정상입니다."
- 대기 중엔 **뭘 기다리는지** 알려주기
  - 예: "지금 Python과 Node.js를 설치 중입니다. 2~5분 걸립니다. 그냥 기다려 주세요."
- 에러 발생 시 **겁먹지 않도록** 안내
  - 예: "빨간 글씨가 나와도 괜찮습니다. 경고일 뿐이에요. 계속 진행합니다."
- 각 단계 완료 시 **칭찬 + 다음 단계 예고**
  - 예: "완료됐습니다! 이제 AI 모델을 연결할게요. (전체 중 3/5 단계)"

### 절대 하지 말 것
- 여러 선택지를 한꺼번에 쏟아내기 (하나씩 순서대로)
- "이건 당연히 아시겠지만..." 식의 표현
- 에러 코드를 설명 없이 그냥 보여주기
- "실패했습니다" 만 출력하고 해결책 없이 끝내기
